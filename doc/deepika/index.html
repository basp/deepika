Deepika

basp

Deepika is a MOO in the spirit of LambdaMOO. MOO stands for MUD object oriented. MUD stands for multi-user domain. It is the predecessor to what we nowadays call an MMO. A MOO is a multi-user domain object oriented environment.

1 Overview

Using the original LambdaMOO as a reference, we can typically divide a MOO into a few key components. At the very core is a TCP server that is not unlike a typical chat server. It reads lines of text and it sends lines of text. Unlike a chat server, a MOO server has a world state which is represented as a database of objects. Clients (i.e. players) are part of this state and represented by objects as well.

And even though nowadays most chat servers will do some amount of parsing or basic text replacement, a MOO has a command parser with a simple but surprisingly powerful syntax and evaluation behavior. In short, it will accept natural language (English) commands and match them to executable code (called verbs) in the object database.

When you consider that this player invoked code can create objects, destroy or relocate them, add properties or verbs (executable code) and manipulate their values, it is apparent that this leads to a very dynamic environment.

2 Reference

This section is an overview of the public interface provided by the modules that make up deepika.

2.1 Database

 (require deepika/db) package: deepika

The database module is responsible for persisting the state of the world. In our case, the world consists mainly of three things: objects, properties and verbs.

value

$nothing : nothing?

Represents the absence of a value and can generally be used as a sensible default for database values.

procedure

(nothing? x) → boolean?

  x : any/c
Returns #t if x is $nothing.

2.1.1 Objects

procedure

(objid? x) → boolean?

  x : any/c
Returns #t if x is an object id and #f otherwise.

Note that the result of this function only confirms the data type. It does not confirm the existance of a valid object. Use valid? for that case instead.

procedure

(objid->number oid) → number?

  oid : objid?
Converts an object id oid into an number.

procedure

(number->objid num) → objid?

  num : integer?
Converts a number num into an object id.

Examples:
> (objid->number (objid 123))

123

> (number->objid 123)

'#s(objid 123)

> (objid->number (number->objid 123))

123

Note that number->objid can be used to create object id’s that are not valid? nor valid+?.

Examples:
> (objid->number $nothing)

-1

> (valid? (number->objid -1))

#f

> (valid+? (number->objid -123))

#f

procedure

(valid? x) → boolean?

  x : any/c
Returns #t if x is an object id that points to an actual object and #f otherwise.

Examples:
> (define oid (number->objid 1))
> (valid? oid)

#f

> (create-object!)

'#s(objid 1)

> (valid? (number->objid 1))

#t

procedure

(valid+? x) → boolean?

  x : any/c
Like valid? but it will also return #t if x is $nothing.

Examples:
> (valid? $nothing)

#f

> (define oid (create-object!))
> (valid? oid)

#t

> (valid? (objid 123))

#f

procedure

(create-object! [oid]) → objid?

  oid : valid+? = $nothing
Create a new object with its parent set to oid. If oid is not specified then the object’s parent property will be set to $nothing.

procedure

(destroy-object! oid) → any/c

  oid : valid?
Destroy the object with id oid. It will no longer be accessible from the database.

procedure

(get-object-name oid) → string?

  oid : valid?
Returns the name of the object with id oid.

procedure

(set-object-name! oid value) → any

  oid : valid?
  value : string?
Sets the name of the object referenced by oid to value.

procedure

(get-parent oid) → valid+?

  oid : valid?
Returns the parent id of the object with id oid.

procedure

(set-parent! oid new-parent) → any

  oid : valid?
  new-parent : valid+?
Sets the parent of object with id oid to the value of new-parent.

procedure

(get-children oid) → (listof valid?)

  oid : valid?
Returns the objects that have oid set as their parent object.

Examples:
> (define o (create-object!))
> (define p (create-object!))
> (get-parent o)

'#s(objid -1)

> (set-parent! o p)
> (get-parent o)

'#s(objid 4)

> (get-children (get-parent o))

'(#s(objid 3))

procedure

(get-location oid) → valid+?

  oid : valid?
Returns the location id of the object specified by oid.

procedure

(set-location! oid new-location) → any

  oid : valid?
  new-location : valid+?
Sets the location of object with id oid to the value of new-location.

2.2 Tasks

 (require deepika/tasks) package: deepika

Note that the tasks module does not actually execute any tasks. Instead, it is merely a storage device and as such, actual execution of the tasks that are queued and ready to run is solely up to the client of this module.

procedure

(task? x) → boolean?

  x : any/c
Returns #t if x is a task structure.

procedure

(task-id x) → integer?

  x : task?
Returns the id of the task given by x.

procedure

(task-th x) → procedure?

  x : task?
Returns the thunk of the task given by x.

procedure

(task/valid? id) → boolean?

  id : integer?
Like valid? but for tasks instead. It will return #t if there is a task with specified id.

procedure

(task-start! del th) → task/valid?

  del : integer?
  th : procedure?
Queues thunk as a new task to start after del seconds have elapsed. The result of this function is the id of the queued task.

procedure

(task-ready? id) → boolean?

  id : task/valid?
Returns #t if the task specified by id is ready to run.

Examples:
> (define tid (task-start! 5 (λ () (displayln "Go!"))))
> tid

1

> (task-ready? tid)

#f

> (sleep 5)
> (task-ready? tid)

#t

procedure

(task-remove! id) → any

  id : task/valid?
Irrevocably removes the task specified by id from the queue. It will not be executed.

procedure

(get-task id) → task?

  id : task/valid?
Returns the task data associated with task id. The result can be queried with task-id and task-proc.

procedure

(tasks) → (listof task/valid?)

Returns a list of ids of all tasks currently on the queue.

Examples:
> (define tid (task-start! 30 (λ () (displayln "Uhmmm..."))))
> tid

2

> (task/valid? tid)

#t

> (tasks)

'(1 2)

> (task-remove! tid)
> (task/valid? tid)

#f

> (tasks)

'(1)

Note that the queue will never remove tasks unless it is explicitly asked to do so. That’s why the latent task of the previous example shows up in this example as well.

procedure

(tasks/ready) → (listof task/valid?)

Like tasks but returns only those tasks that are ready to run (i.e. when task-ready? returns #t).

2.3 Parser

 (require deepika/parser) package: deepika

procedure

(parse/args s) → (listof string?)

  s : string?
Parses a string into arguments. This tokenizes the string on whitespace and treats quoted strings as one token.

For example, the result of the following expression is #t:

(equal? (parse/args "foo \"bar quux\" frotz")
        (list "foo" "bar quux" "frotz"))
